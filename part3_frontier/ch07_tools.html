Chapter 7 · Part III Tools & Reproducibility

Every result in this book is a computation, and every computation is public. This chapter is the recipe: the engine, the commands that produced each closure, and the standing invitation to push the open doors yourself.

7.1   The engine

All of it lives in orbit-gen — an orbit-matrix enumerator, a SAT-modulo-symmetries (SMS) solver wrapping CaDiCaL with a nauty canonicity propagator, a partial-difference -set search, and a Gram-PSD / Euclidean filter. The two entry points used here are the search binary (prescribed-automorphism orbit matrices) and the prescribed-difference-set / Gram tools.

7.2   Reproduce the symmetry closures

# No order-3 automorphism — orbit matrices over all fixed-point counts
search 99 14 1 2 3 --f <f> --timeout 3600        # every f → 0 matrices

# No order-11 automorphism — the three semiregular configurations
search 99 14 1 2 11 --semiregular --timeout 3600  # all UNSAT

# Not an abelian Cayley graph — no (99,14,1,2) difference set
pds_cnf 99 14 1 2 --group Z99            | kissat   # UNSAT
pds_cnf 99 14 1 2 --group Z3xZ33         | kissat   # UNSAT

Each "$0$ matrices" / "UNSAT" is an EXHAUSTED certificate — a real theorem (Appendix B.4), not a timeout.

7.3   The order-7 door

The unique Cesarz–Woldar configuration $[1,7^{14}]$ needs the full-diagonal orbit-matrix path (a subtle diagonal-valency case that earlier builds got wrong — see the companion notes). It does not finish on a desktop: 

# The famous open door: 1 fixed point, 14 orbits of size 7
ORBIT_FULL_DIAG=1 search 99 14 1 2 7 --f 1 --timeout 0
#   → ~281 M nodes in 1 h, stack depth still climbing — cluster-scale.
#   A lifted matrix wins Conway's prize; UNSAT strengthens Cesarz–Woldar.

7.4   The SAT and triangle encodings

# Direct CNF / pseudo-Boolean encodings (Chapter 4) — do not decide
conway99-encode --cnf  > conway99.cnf      # ~7e8 clauses, 22 GB
conway99-encode --pb   > conway99.opb      # roundingsat
conway99-encode --triangular > tri231.opb  # the (231,18,5,≤3) view

7.5   The Gram-PSD / Euclidean filter

# Sound rank-44 PSD pruning on the (-4)-eigenspace embedding (Chapter 5)
conway99-extension --gram --rank 44 --timeout 3600
#   sound: never rejects a real graph; cluster-scale to exhaust.
GPU_GRAM_PSD=1 ...    # batched Cholesky on a Vulkan GPU for the inner loop

7.6   The standing invitation

Push a door, claim a result

Drive the order-$7$ enumeration ($\S$7.3) to completion on a cluster, or complete the involution sweep ($46$ configurations via search 99 14 1 2 2 --f <f>), or run the claw-minimisation / Gram search to exhaustion. A single lifted matrix is Conway's $\$1000$ graph; a full UNSAT is a non-existence theorem. Either way you will have moved the most famous open door in the strongly-regular tables — which is the whole point of writing the recipe down.
